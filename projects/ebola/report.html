Comparative Ebola Virus Genomics

My goal is to analyze ebola virus sequencing data sets collected at three different time points from the 2014 outbreak. I also want to compare different aligners: BWA, Smalt, and Subread.

Collection Dates and Accessions:

  • May 28th - SRR1553468
  • July 8th - SRR1972721
  • August 20th - SRR1735146

That information can be found here.

The reference genome I’ll be comparing these samples to is the longest assembled isolate of Gire et al. 2014, G3686 (accesion KM034562.1).

Contamination

The ebola samples were isolated from human hosts meaning that some if not most of the reads within our fastq files will be of human origin considering how much larger the human genome is as compared to the ebola genome (5 magnitudes smaller).

Alignments (Units 7 & 8)

I decided to use BWA MEM to align my reads to my reference for each sample and I’ll show the basic alignment stats below. The command was:

bwa mem -t 4 refs/ebola-2014.fasta unit7/data/${month}_2014/trimmed_1.fastq.gz unit7/data/${month}_2014/trimmed_2.fastq.gz | samtools view -b - > unit7/data/${month}_2014/aln.bam

Variant Analysis (Unit 9)

Using samtools and bcftools I called for SNPs within each sample independently using the followin command:

$ samtools mpileup -ugf ../../../refs/ebola-2014.fasta sorted.bam | bcftools call -cv > results.vcf

Overview

Annotation (Unit 10)

In order to annotate those SNP calls I decided to utilize a simple approach and see in which region the SNPs fell. That is, whether they intersected with genomic intervals or intergenic regions of the genome.

In order to create an interval file that bedtools likes to work with I had to first download the GenBank file and convert it to GFF3 using Seqret of the EMBOSS suite.

$ efetch -db nucleotide -id KM034562 -format gb > ebola-2014.gb

I converted the VCF file to bed using the following command:

$ awk '! /\#/' results.vcf | awk '{if(length($4) > length($5)) print $1"\t"($2-1)"\t"($2+length($4)-1); else print $1"\t"($2-1)"\t"($2+length($5)-1)}' > variants.bed

And then used the following bedtools command to investigate where those variants overlapped:
